How Menus Work in the CQL Propel Theme

Understanding how menus are assigned and rendered in the CQL Propel theme is crucial for effective navigation management:

Menu Assignment Process
  1. Create Menu in Shopify Admin: Navigate to Online Store > Navigation and create a new menu or edit existing ones
  2. Assign to Theme Location: In the theme editor (Online Store > Themes > Customize), go to Theme Settings > Navigation
  3. Select Menu Handle: Choose which menu to display in each navigation location (header, footer, etc.)
  4. Save Theme Settings: Changes must be saved in the theme editor to take effect
Menu Location Mapping

The CQL Propel theme uses specific menu handles that must match exactly:

  • main-menu → Primary header navigation (desktop and mobile)
  • utility-menu → Header utility links (account, cart, search)
  • shop → Footer product categories column
  • customer-care → Footer support links column
  • about-us → Footer company information column
  • copyright → Footer bottom legal links
Menu Structure Requirements
  • Maximum Depth: 2 levels (main item + sub-item)
  • Mobile Behavior: Main menu collapses into hamburger menu on mobile
  • Utility Menu: Always displays horizontally in header, doesn't collapse
  • Footer Menus: Display as simple vertical lists, no sub-navigation
Common Menu Issues & Solutions
  • Menu Not Appearing: Check that the menu handle matches exactly in theme settings
  • Wrong Menu Displayed: Verify menu assignment in Theme Settings > Navigation
  • Mobile Menu Issues: Ensure main-menu has proper structure for mobile collapse
  • Footer Menu Problems: Footer menus are simple lists - avoid complex nesting

How to Edit Navigation

  1. Navigate to Online Store > Navigation in Shopify Admin
  2. Select the menu you want to edit
  3. Click Add menu item to add new links
  4. Enter Name (link text) and Link (destination URL or resource)
  5. Drag and drop menu items to reorder
  6. Indent menu items to create sub-navigation (2 levels supported)
  7. Click Save menu
  8. Verify in Theme Editor: Check that the menu is properly assigned in Theme Settings > Navigation

Product & Merchandising

Product Setup Guide

This guide covers the complete process for setting up new products in the Superfeet platform. Since product data is managed manually (no PIM integration), following these procedures ensures consistency and proper functionality across all storefronts.

Step 1: Basic Product Information

Start with the essential product details:

  1. Navigate to Products > Add Product
  2. Product Title: Use descriptive, SEO-friendly names (e.g., "All-Purpose Cushion Medium Arch")
  3. Product Description: Include key features, benefits, and usage information
  4. Product Type: Categorize appropriately (e.g., "Insole", "Cushion", "Support")
  5. Vendor: Set to "Superfeet" for all products
  6. Tags: Add relevant tags for filtering and collections

Step 2: Product Images

Upload high-quality product images following these specifications:

  • Primary Image: High-resolution product shot on white background (minimum 1024x1024px)
  • Secondary Images: Multiple angles, lifestyle shots, size comparisons
  • Alt Text: Descriptive alt text for accessibility
  • File Format: Optimized JPEG or WebP for web performance

Step 3: Variant Setup

Create product variants with proper naming conventions:

  1. Option Name: Set to "Size" for all products
  2. Option Values: Use appropriate naming convention based on product type:
    • Unisex Products: "Women 6.5 - 8 | Men 5.5 - 7"
    • Women's Products: "Women 6 - 8", "Women 7.5 - 9.5"
    • Men's Products: "Men 7 - 9", "Men 8.5 - 10.5"
  3. SKU: Use Superfeet internal SKU format (e.g., FL143306)
  4. Barcode: Enter UPC code if available

Step 4: Pricing & Inventory

Set up pricing and inventory for each variant:

  • Price: Set retail price for each variant
  • Compare at Price: Use for sale pricing when applicable
  • Cost per Item: Add COGS data (important for financial reporting)
  • Inventory: Set initial inventory quantities
  • Track Quantity: Enable for all variants

Step 5: Product Metafields

Add essential metafields for theme functionality (see detailed metafield documentation below)

Step 6: SEO & Publishing

Complete SEO setup and publish the product:

  • Page Title: SEO-optimized title
  • Meta Description: Compelling product description for search results
  • URL Handle: Clean, descriptive URL
  • Publish Status: Set to "Draft" for review or "Active" to go live

Bundle Setup

Superfeet offers two types of product bundles that provide customers with value and convenience. Understanding the different bundle types and their inventory requirements is essential for proper setup and management.

Bundle Types

Fixed Bundles

Fixed bundles contain predetermined products with specific quantities. These bundles have their own inventory that must be managed separately from individual products.

  • Inventory Management: Each fixed bundle has its own SKU and inventory level
  • Setup Location: Created and managed in ReCharge app
  • Inventory Tracking: Bundle inventory is tracked independently of component products
  • Example: "Work Bundle" containing 1 pair of work insoles + 1 pair of casual insoles
Flex Bundles (Custom Bundles)

Flex bundles allow customers to choose from a selection of products to create their own bundle. These bundles pull inventory from the individual products that make up the bundle.

  • Inventory Management: Uses inventory from individual component products
  • Setup Location: Created and managed in ReCharge app
  • Inventory Tracking: No separate bundle inventory - relies on component product availability
  • Example: "Custom Arch Support Bundle" where customer selects 2 products from arch support category

Setting Up Bundles in ReCharge

  1. Navigate to ReCharge app in Shopify admin
  2. Go to Products > Bundles
  3. Click Create Bundle
  4. Choose Bundle Type:
    • Fixed Bundle: Select specific products and quantities
    • Flex Bundle: Define product categories and selection rules
  5. Configure Bundle Details:
    • Bundle name and description
    • Pricing and discount structure
    • Product selection (fixed) or category rules (flex)
    • Inventory settings (fixed bundles only)
  6. Set Inventory Levels (Fixed Bundles Only):
    • Define initial inventory quantity for the bundle
    • Set low stock alerts if needed
    • Configure inventory tracking settings
  7. Test Bundle Functionality:
    • Verify bundle appears correctly on storefront
    • Test purchase flow and inventory deduction
    • Confirm pricing and discount calculations
  8. Publish Bundle when ready to go live

Bundle Inventory Management

Fixed Bundle Inventory
  • Separate Tracking: Fixed bundles have independent inventory from component products
  • Manual Updates: Update inventory levels in ReCharge when needed
  • Stock Alerts: Set up low stock notifications for bundle-specific inventory
  • Bundle Management: Can unpublish or remove bundles without affecting individual products
Flex Bundle Inventory
  • Component-Based: Inventory depends on availability of individual products
  • Real-Time Availability: Bundle availability updates automatically based on component stock
  • No Separate Management: No need to manage bundle-specific inventory levels
  • Component Changes: Adding/removing products from categories affects bundle availability

Bundle Best Practices

  • Clear Naming: Use descriptive names that indicate bundle type and contents
  • Value Proposition: Ensure bundles provide clear value to customers
  • Inventory Planning: For fixed bundles, plan inventory based on expected demand
  • Regular Review: Monitor bundle performance and adjust as needed
  • Testing: Always test bundle functionality before launching

Inventory Management

Inventory management at Superfeet is straightforward and manually controlled. The team manages inventory levels directly in Shopify, with minimal automated integration to external systems.

Inventory Management Overview

  • Manual Control: All inventory levels are managed by the Superfeet team in Shopify admin
  • Macola Integration: Macola pulls new orders and pushes fulfillment updates to Shopify
  • No Real-Time Sync: Inventory levels are not automatically synchronized between systems
  • High Stock Levels: Most products start with 10,000 units to minimize out-of-stock situations

Inventory Setup

When setting up new products:

  • Initial Stock: Set to 10,000 units for most products
  • Track Quantity: Enabled for all product variants
  • Allow Backorders: Disabled (products show as sold out when inventory reaches zero)
  • Inventory Policy: Set to "Deny" to prevent overselling

Inventory Updates

To update inventory levels:

  1. Navigate to Products > [Product Name]
  2. Scroll to Variants section
  3. Click on variant to expand details
  4. Update Quantity on hand field
  5. Click Save to update the variant

Bulk Inventory Updates

For updating multiple products, use Matrixify:

  1. Export products with inventory data
  2. Update Quantity on hand values in spreadsheet
  3. Import updated data back to Shopify

Low Stock Management

  • Rare Occurrence: Products rarely go out of stock due to high initial inventory levels
  • Manual Monitoring: Team monitors inventory levels through Shopify admin
  • Restocking: When needed, inventory is updated manually based on sales data
  • Product Removal: When products are no longer sold, they are typically unpublished or removed from collections

Inventory Reporting

  • Shopify Reports: Use built-in inventory reports to track stock levels
  • Sales Analysis: Monitor product performance to inform inventory decisions
  • Macola Data: Use fulfillment data from Macola to verify inventory accuracy
Note: The high initial inventory levels (10,000 units) were chosen to minimize out-of-stock situations. This approach works well for Superfeet's product catalog and sales patterns.

Cost of Goods Sold (COGS) Data

COGS data is critical for Superfeet's financial operations and must be included in every product setup. Since Superfeet manages product data manually (no PIM or automated system integration), proper COGS documentation and procedures are essential for accurate financial reporting and profitability analysis.

Why COGS Data is Critical

COGS data enables Superfeet to:

  • Profitability Analysis: Track margins and profitability by product and variant
  • Financial Reporting: Generate accurate cost reports for accounting and tax purposes
  • Pricing Strategy: Make informed pricing decisions based on actual product costs
  • Inventory Valuation: Calculate inventory value using cost basis rather than retail pricing
  • Margin Tracking: Monitor gross margins across different product categories and regions
  • Regional Pricing: Adjust pricing strategies for different markets based on cost structures
Initial Product Setup - COGS Requirements

Every new product must include COGS data during initial setup:

  1. Navigate to Products > Add Product
  2. Complete basic product information (title, description, images, etc.)
  3. Create product variants with proper size naming conventions
  4. For each variant, add COGS data:
    • Click on the variant to expand details
    • Enter Cost per item in the "Pricing" section
    • Ensure cost is entered in the correct currency for the store
    • Verify cost accuracy with product team or supplier data
  5. Save the product with all COGS data included
Critical: Never publish a product without COGS data. This data is required for financial reporting and margin analysis. Products without COGS data will cause issues with accounting systems and financial reporting.
Accessing and Editing COGS Data

Individual Product Updates:

  1. Navigate to Products > [Product Name]
  2. Scroll to the Variants section
  3. Click on the specific variant to expand details
  4. Edit the Cost per item field in the Pricing section
  5. Click Save to update the variant
Bulk COGS Management with Matrixify

For updating COGS data across multiple products or variants, use Matrixify for efficient bulk operations:

Export Current COGS Data
  1. Open Matrixify app in Shopify admin
  2. Select Export from the main menu
  3. Choose Products as the data type
  4. In the export settings:
    • Select All products or specific collections
    • Include Variants in the export
    • Ensure Cost per item is included in the export fields
  5. Click Export and download the CSV file
Update COGS Data in Spreadsheet
  1. Open the exported CSV file in Excel or Google Sheets
  2. Locate the Variant Cost column
  3. Update cost values for the products/variants you need to modify
  4. Ensure cost values are numeric (no currency symbols)
  5. Verify all required fields are populated
  6. Save the file as CSV format
Import Updated COGS Data
  1. Return to Matrixify app
  2. Select Import from the main menu
  3. Upload your updated CSV file
  4. Choose Update existing products as the import mode
  5. Map the Variant Cost field to the correct Shopify field
  6. Review the import preview to ensure data accuracy
  7. Click Import to update the products
Matrixify Best Practice: Always export current data before making bulk changes. This provides a backup and allows you to verify the current state before making updates. Test bulk imports on a small subset of products first.
COGS Data Validation

After updating COGS data (individually or in bulk), validate the changes:

  • Spot Check: Verify a sample of updated products show correct COGS values
  • Margin Verification: Check that calculated margins make sense (price - cost = margin)
  • Currency Consistency: Ensure all COGS values are in the correct currency for each store
  • Accounting Sync: Verify that updated COGS data syncs properly with accounting systems
Common COGS Management Scenarios
  • New Product Launch: Include COGS data in initial product setup
  • Cost Changes: Update COGS when supplier costs change
  • Currency Fluctuations: Adjust COGS for regional stores when exchange rates change
  • Seasonal Updates: Bulk update COGS for seasonal product lines
  • Product Discontinuation: Maintain COGS data for discontinued products for historical reporting

Discount Management

Superfeet uses Regios Discounts app (installed on US store only) for advanced discount logic beyond Shopify's native discounts.

Discount Types

Type Use Case Method
Percentage Off 25% off entire order, 20% off specific collection Code or Automatic
Fixed Amount Off $10 off $50+, $20 off $100+ Code or Automatic
Free Shipping Free shipping on all orders Code or Automatic
Buy X Get Y Buy 2 get 1 free, Buy 3 get 20% off Code or Automatic

How to Create a Discount Code

  1. Navigate to Discounts in Shopify Admin
  2. Click Create discount
  3. Choose discount type (Amount off, Percentage off, Free shipping, Buy X Get Y)
  4. Choose discount method (Code or Automatic)
  5. Configure discount details:
    • Discount code or title
    • Value (percentage or dollar amount)
    • Applies to (entire order, specific collections, specific products)
    • Minimum purchase requirements
    • Customer eligibility (all customers, specific segments)
    • Usage limits (total uses, one per customer)
    • Active dates (start and end date/time)
  6. Click Save

Discount Best Practices

  • Naming Convention: Use descriptive codes (SPRING25, FREESHIP50, WELCOME10)
  • Expiration Dates: Always set end dates to prevent indefinite promotions
  • Usage Limits: Set limits to control budget exposure
  • Test Before Launch: Place test orders to verify discount logic
  • Communicate Clearly: Ensure marketing materials match discount restrictions
  • Track Performance: Monitor discount usage and impact on margins
Regios vs. Shopify Discounts: Regios Discounts provides more sophisticated discount logic (stacking, tiered discounts, complex conditions). For advanced promotions, use Regios. For simple discounts, use Shopify native discounts.

Troubleshooting & Support

Common Issues & Solutions

Product Not Appearing in Collection

Symptoms: Product is published but doesn't show in expected collection.

Solutions:

  1. Verify product has correct tags for smart collection rules
  2. Check product is published to "Online Store" sales channel
  3. For custom collections, manually add product to collection
  4. Confirm collection is published and visible
  5. Check if SearchSpring (Athos Commerce) indexing is up to date

Discount Code Not Working

Symptoms: Customer reports discount code shows error at checkout.

Solutions:

  1. Verify discount code is active and not expired
  2. Check minimum purchase requirements are met
  3. Confirm customer eligibility (all customers vs. specific segments)
  4. Verify products in cart are eligible for discount
  5. Check if another discount is already applied (stacking rules)
  6. Review usage limits (total uses or per customer)

SearchSpring (Athos Commerce) Not Displaying Correctly

Symptoms: Product listing page shows Shopify native search instead of SearchSpring (Athos Commerce).

Solutions:

  1. Check SearchSpring (Athos Commerce) app is installed and active
  2. Verify theme setting plp_search_appearance is set to "searchspring"
  3. Confirm SearchSpring (Athos Commerce) account is active and site is indexed
  4. Clear browser cache and test in incognito mode
  5. Check browser console for JavaScript errors

Inventory Sync Issues

Symptoms: Inventory levels incorrect or not updating from Macola ERP.

Solutions:

  1. Contact technical team to verify ERP sync status
  2. Check if manual inventory adjustments caused conflict
  3. Verify product SKUs match between Shopify and Macola
  4. Review sync logs for error messages
  5. Avoid manual inventory changes unless coordinated with ERP team

Theme Sections Not Saving

Symptoms: Changes in theme editor don't persist after saving.

Solutions:

  1. Ensure you're editing the published theme (not a draft)
  2. Check for JavaScript errors in browser console
  3. Try saving in a different browser
  4. Verify you have proper user permissions
  5. Contact Shopify Support if issue persists
