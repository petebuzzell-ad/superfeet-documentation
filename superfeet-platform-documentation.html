Platform Architecture

Multi-Region Structure

Superfeet operates on a multi-store Shopify Plus architecture with separate Shopify stores for each region. This approach was required during the Magento to Shopify migration to maintain proper TLD (top-level domain) structure for regional markets.

Migration Context: Prior to October 16, 2024, Superfeet.com was a unified Magento multi-region site serving US, Canada, UK, EU, and Australia. Shopify's architecture required splitting into separate stores by region.

Store Architecture

Store Shopify Subdomain Status Backend System
US (superfeetww) superfeetww.myshopify.com Active - Primary Macola ERP Integration
Canada (superfeet-ca) superfeet-ca.myshopify.com Active Macola ERP Integration
UK (superfeet-uk) superfeet-uk.myshopify.com Active Macola ERP Integration

Geographic Redirection

The Geo:Pro Geolocation app is installed on the US site (superfeetww) to automatically redirect international visitors to the appropriate regional storefront based on their IP address location.

How Menus Work in the CQL Propel Theme

Understanding how menus are assigned and rendered in the CQL Propel theme is crucial for effective navigation management:

Menu Assignment Process
  1. Create Menu in Shopify Admin: Navigate to Online Store > Navigation and create a new menu or edit existing ones
  2. Assign to Theme Location: In the theme editor (Online Store > Themes > Customize), go to Theme Settings > Navigation
  3. Select Menu Handle: Choose which menu to display in each navigation location (header, footer, etc.)
  4. Save Theme Settings: Changes must be saved in the theme editor to take effect
Menu Location Mapping

The CQL Propel theme uses specific menu handles that must match exactly:

  • main-menu → Primary header navigation (desktop and mobile)
  • utility-menu → Header utility links (account, cart, search)
  • shop → Footer product categories column
  • customer-care → Footer support links column
  • about-us → Footer company information column
  • copyright → Footer bottom legal links
Menu Structure Requirements
  • Maximum Depth: 2 levels (main item + sub-item)
  • Mobile Behavior: Main menu collapses into hamburger menu on mobile
  • Utility Menu: Always displays horizontally in header, doesn't collapse
  • Footer Menus: Display as simple vertical lists, no sub-navigation
Common Menu Issues & Solutions
  • Menu Not Appearing: Check that the menu handle matches exactly in theme settings
  • Wrong Menu Displayed: Verify menu assignment in Theme Settings > Navigation
  • Mobile Menu Issues: Ensure main-menu has proper structure for mobile collapse
  • Footer Menu Problems: Footer menus are simple lists - avoid complex nesting

How to Edit Navigation

  1. Navigate to Online Store > Navigation in Shopify Admin
  2. Select the menu you want to edit
  3. Click Add menu item to add new links
  4. Enter Name (link text) and Link (destination URL or resource)
  5. Drag and drop menu items to reorder
  6. Indent menu items to create sub-navigation (2 levels supported)
  7. Click Save menu
  8. Verify in Theme Editor: Check that the menu is properly assigned in Theme Settings > Navigation

Product & Merchandising

Product Catalog

Superfeet's product catalog consists of active products, primarily insoles and footcare accessories. Products are highly structured with extensive metafields to support the Insole Finder quiz, product comparison tools, and detailed product pages.

Product Categories

  • Insoles: All-Purpose, Run, Casual, Work, Sport-specific
  • Arch Height Variations: Low, Medium, High arch support options
  • Specialty Products: Women's specific, Wide fit, Kids sizing
  • Bundles: Product bundles via Recharge app
  • Accessories: Complementary footcare products

Product Naming Convention

Products follow a structured naming pattern:

  • all-purpose-cushion - Base product name
  • all-purpose-support-medium-arch - Product with arch specification
  • all-purpose-womens-high-impact-support - Gender and feature specific
  • run-cushion-medium-arch - Activity-specific with arch type

Variant Structure

Products use size-based variants following this pattern:

  • Option Name: Size
  • Option Values: "Women 6.5 - 8 | Men 5.5 - 7" (gender-specific size ranges)
  • SKU Format: FL143306 (Superfeet internal SKU)
  • Barcode: UPC format (086301801331)

Cost of Goods Sold (COGS) Data

Shopify stores comprehensive COGS data for each product variant, which is essential for:

  • Profitability Analysis: Track margins and profitability by product and variant
  • Financial Reporting: Generate accurate cost reports for accounting and tax purposes
  • Pricing Strategy: Make informed pricing decisions based on actual product costs
  • Inventory Valuation: Calculate inventory value using cost basis rather than retail pricing
  • Margin Tracking: Monitor gross margins across different product categories and regions

Access COGS Data: Navigate to Products > [Product] > Variants to view and edit cost per item for each variant. This data syncs with accounting systems and is used for financial reporting and margin analysis.

Product Tags

Products use extensive tagging for filtering and categorization:

  • Gender: MEN'S, WOMEN'S, UNISEX, KIDS
  • Activity: WALKING, RUNNING, CASUAL, WORK, BASEBALL, BASKETBALL, SOCCER, etc.
  • Product Type: INSOLES, ALL-PURPOSE, SPORT-SPECIFIC
  • Marketing: BEST SELLERS, NEW, discount-eligable
  • Collections: Tags map to smart collection rules

Inventory Management

Superfeet inventory is managed across three warehouse locations, with inventory synced from the Macola ERP system (inherited from Magento integration).

Warehouse Locations

Location Code Purpose
Superfeet Worldwide LLC Primary Main fulfillment center
Quiet Atlanta Warehouse ATL Secondary distribution
Quiet La Palma Warehouse LAP West coast distribution

Inventory Tracking

Each product variant tracks inventory across all three locations:

  • Available: Units available for sale
  • On Hand: Physical units in warehouse
  • Committed: Units allocated to orders
  • Reserved: Units on hold
  • Incoming: Units expected from suppliers
  • Damaged: Units marked as damaged
  • Quality Control: Units in QC process
  • Safety Stock: Minimum inventory threshold

Inventory Sync Process

Inventory levels sync from Macola ERP to Shopify. The legacy integration from Magento days uses:

  • Scheduled sync jobs to push inventory updates
  • Real-time deduction when orders are placed
  • Inventory reconciliation to prevent overselling
ERP Integration: The Macola ERP integration documentation from the Magento era provides context for inventory sync, order processing, and shipment tracking. See artifacts/README.txt for technical details of the legacy integration.

Low Stock Thresholds

Products can set custom low stock thresholds via metafield:

  • cql.low_stock_threshold - Integer value triggering low stock alerts

How to View Inventory

  1. Navigate to Products > Inventory in Shopify Admin
  2. Use filters to narrow by product, location, or availability
  3. Click on a product to see detailed inventory by variant and location
  4. Use Adjust inventory to manually update quantities (if needed)
Manual Adjustments: Avoid manual inventory adjustments unless coordinated with the ERP system. Manual changes can cause sync conflicts and overselling issues.

Discount Management

Superfeet uses Regios Discounts app (installed on US store only) for advanced discount logic beyond Shopify's native discounts.

Discount Types

Type Use Case Method
Percentage Off 25% off entire order, 20% off specific collection Code or Automatic
Fixed Amount Off $10 off $50+, $20 off $100+ Code or Automatic
Free Shipping Free shipping on all orders Code or Automatic
Buy X Get Y Buy 2 get 1 free, Buy 3 get 20% off Code or Automatic

How to Create a Discount Code

  1. Navigate to Discounts in Shopify Admin
  2. Click Create discount
  3. Choose discount type (Amount off, Percentage off, Free shipping, Buy X Get Y)
  4. Choose discount method (Code or Automatic)
  5. Configure discount details:
    • Discount code or title
    • Value (percentage or dollar amount)
    • Applies to (entire order, specific collections, specific products)
    • Minimum purchase requirements
    • Customer eligibility (all customers, specific segments)
    • Usage limits (total uses, one per customer)
    • Active dates (start and end date/time)
  6. Click Save

Discount Best Practices

  • Naming Convention: Use descriptive codes (SPRING25, FREESHIP50, WELCOME10)
  • Expiration Dates: Always set end dates to prevent indefinite promotions
  • Usage Limits: Set limits to control budget exposure
  • Test Before Launch: Place test orders to verify discount logic
  • Communicate Clearly: Ensure marketing materials match discount restrictions
  • Track Performance: Monitor discount usage and impact on margins
Regios vs. Shopify Discounts: Regios Discounts provides more sophisticated discount logic (stacking, tiered discounts, complex conditions). For advanced promotions, use Regios. For simple discounts, use Shopify native discounts.

Troubleshooting & Support

Common Issues & Solutions

Product Not Appearing in Collection

Symptoms: Product is published but doesn't show in expected collection.

Solutions:

  1. Verify product has correct tags for smart collection rules
  2. Check product is published to "Online Store" sales channel
  3. For custom collections, manually add product to collection
  4. Confirm collection is published and visible
  5. Check if SearchSpring (Athos Commerce) indexing is up to date

Discount Code Not Working

Symptoms: Customer reports discount code shows error at checkout.

Solutions:

  1. Verify discount code is active and not expired
  2. Check minimum purchase requirements are met
  3. Confirm customer eligibility (all customers vs. specific segments)
  4. Verify products in cart are eligible for discount
  5. Check if another discount is already applied (stacking rules)
  6. Review usage limits (total uses or per customer)

SearchSpring (Athos Commerce) Not Displaying Correctly

Symptoms: Product listing page shows Shopify native search instead of SearchSpring (Athos Commerce).

Solutions:

  1. Check SearchSpring (Athos Commerce) app is installed and active
  2. Verify theme setting plp_search_appearance is set to "searchspring"
  3. Confirm SearchSpring (Athos Commerce) account is active and site is indexed
  4. Clear browser cache and test in incognito mode
  5. Check browser console for JavaScript errors

Inventory Sync Issues

Symptoms: Inventory levels incorrect or not updating from Macola ERP.

Solutions:

  1. Contact technical team to verify ERP sync status
  2. Check if manual inventory adjustments caused conflict
  3. Verify product SKUs match between Shopify and Macola
  4. Review sync logs for error messages
  5. Avoid manual inventory changes unless coordinated with ERP team

Theme Sections Not Saving

Symptoms: Changes in theme editor don't persist after saving.

Solutions:

  1. Ensure you're editing the published theme (not a draft)
  2. Check for JavaScript errors in browser console
  3. Try saving in a different browser
  4. Verify you have proper user permissions
  5. Contact Shopify Support if issue persists
